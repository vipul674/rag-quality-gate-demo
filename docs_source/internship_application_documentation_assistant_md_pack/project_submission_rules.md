# Project Submission Rules

## Purpose

Project submissions help NovaTech evaluate practical ability, execution quality, and role fit.

## Who Must Submit Projects

Project submission is required for:

- Software Engineering applicants
- AI Engineering applicants
- Data Science applicants
- Cybersecurity applicants
- Product Design applicants

## Minimum Project Requirement

Applicants must submit at least **one relevant project**.

Applicants may submit up to **three projects**.

## Accepted Project Types

Accepted projects include:

- Web applications
- Mobile applications
- AI or ML projects
- Data analysis notebooks
- Dashboards
- Security labs or writeups
- Open-source contributions
- Design case studies
- Research prototypes
- API-based tools

## Required Project Information

Each project submission must include:

1. Project title
2. Short problem statement
3. Technologies used
4. Applicant's personal contribution
5. GitHub, demo, notebook, report, or portfolio link
6. Setup instructions, if applicable
7. Known limitations
8. Future improvements

## GitHub Repository Rules

If a GitHub repository is submitted, it should include:

- Clear README
- Installation or usage instructions
- Meaningful commit history
- Organized folder structure
- No exposed API keys
- No private credentials
- No plagiarized code
- License information, if relevant

## Demo Rules

If a live demo is submitted:

- It must be accessible during the review period.
- It should not require paid access.
- It should not expose private user data.
- It should include test credentials if login is required.
- It should not contain offensive or unsafe content.

## AI Project Requirements

AI or LLM projects should explain:

- Model or API used
- Input and output format
- Evaluation method
- Failure cases
- Data source
- Safety or misuse considerations
- Cost or latency tradeoffs, if relevant

## Data Science Project Requirements

Data science projects should include:

- Dataset source
- Data cleaning steps
- Exploratory analysis
- Model or statistical method
- Evaluation metrics
- Visualizations
- Key findings

## Product Design Project Requirements

Design submissions should include:

- Problem statement
- Target users
- User research summary
- Wireframes or prototypes
- Design decisions
- Usability considerations
- Final screens or flows

## Prohibited Submissions

Applicants must not submit:

- Projects copied entirely from tutorials
- Projects they cannot explain
- Code containing secrets or tokens
- Work owned by another person without permission
- Malware or exploit code targeting real systems
- Private company work without approval
- Fake demos or fake metrics

## Review Criteria

Projects are evaluated based on:

| Criterion | Weight |
|---|---:|
| Relevance to selected track | 25% |
| Technical depth | 25% |
| Execution quality | 20% |
| Original contribution | 15% |
| Documentation clarity | 10% |
| Responsible implementation | 5% |
