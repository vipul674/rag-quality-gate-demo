# Common Mistakes

## Purpose

This document lists common mistakes applicants make during the NovaTech Summer Internship Program 2026 application process.

## Resume Mistakes

Common resume mistakes include:

- Uploading a DOCX file instead of a PDF.
- Submitting a resume longer than the allowed limit.
- Missing graduation year.
- Missing email or phone number.
- Listing skills the applicant cannot explain.
- Describing projects vaguely.
- Forgetting GitHub, demo, portfolio, or report links.
- Using too much visual formatting that breaks resume parsing.
- Using an unprofessional file name.
- Including sensitive identity numbers.

## Application Form Mistakes

Common application form mistakes include:

- Selecting the wrong internship track.
- Entering an incorrect email address.
- Submitting duplicate applications.
- Leaving required short answers blank.
- Adding broken project links.
- Providing inconsistent academic details.
- Missing the deadline because of last-minute submission.

## Project Submission Mistakes

Common project mistakes include:

- Submitting copied tutorial projects.
- Not including a README file.
- Exposing API keys or credentials.
- Providing private GitHub links without access.
- Submitting projects unrelated to the selected track.
- Claiming team work as individual work without explanation.
- Not explaining the applicant's personal contribution.

## Assessment Mistakes

Common assessment mistakes include:

- Starting too close to the deadline.
- Ignoring instructions.
- Switching tabs when prohibited.
- Not testing system requirements beforehand.
- Spending too much time on one question.
- Submitting without reviewing answers.

## Interview Mistakes

Common interview mistakes include:

- Not knowing the resume well.
- Not being able to explain project architecture.
- Giving memorized answers.
- Not asking clarifying questions.
- Exaggerating experience.
- Joining late.
- Keeping the resume and project links inaccessible.
- Not preparing examples from past work.

## Communication Mistakes

Common communication mistakes include:

- Sending support requests from a different email than the application email.
- Not including application ID in support emails.
- Sending repeated duplicate emails.
- Using vague subject lines.
- Escalating before waiting for the standard response time.

## Best Practices

Applicants should:

- Submit early.
- Keep documents ready before applying.
- Use one consistent email address.
- Verify every link.
- Practice explaining their projects.
- Review eligibility before applying.
- Keep screenshots of final submission confirmation.
