# Interview Preparation

## Interview Purpose

The NovaTech interview process evaluates whether the applicant can think clearly, communicate effectively, and execute real work during the internship.

## Interview Format

Most applicants will attend one interview of **30 to 45 minutes**.

Some applicants may attend a second interview if:

- The role is highly competitive.
- The first interview score is borderline.
- The hiring team needs deeper project validation.
- The applicant applied to more than one track.

## Interview Areas

Applicants may be evaluated on:

1. Fundamentals
2. Project explanation
3. Problem-solving
4. Communication
5. Learning ability
6. Collaboration
7. Internship motivation

## Technical Interview Expectations

### Software Engineering

Applicants should prepare:

- Arrays and strings
- Hash maps
- Recursion basics
- Sorting and searching
- Time and space complexity
- REST APIs
- Git basics
- Database fundamentals

### AI Engineering

Applicants should prepare:

- Python basics
- Machine learning concepts
- Evaluation metrics
- Prompt engineering
- Embeddings
- Vector search
- APIs
- Limitations of LLMs
- Responsible AI basics

### Data Science

Applicants should prepare:

- SQL queries
- Joins and aggregations
- Data cleaning
- Exploratory data analysis
- Probability and statistics
- Model evaluation
- Charts and interpretation

### Cybersecurity

Applicants should prepare:

- HTTP basics
- Authentication and authorization
- OWASP Top 10 basics
- Linux commands
- Networking fundamentals
- Responsible disclosure

### Product Design

Applicants should prepare:

- Design thinking
- User research
- Wireframes
- Usability testing
- Portfolio walkthrough
- Accessibility basics

## Project Deep-Dive

Applicants should be able to explain:

- Why they built the project
- What problem it solves
- Architecture or design flow
- Technologies used
- Tradeoffs made
- Bugs or failures encountered
- What they would improve next
- Which parts they personally implemented

## Behavioral Questions

Common behavioral questions include:

- Tell us about yourself.
- Why do you want this internship?
- Describe a project you are proud of.
- Tell us about a time you failed.
- How do you handle unclear requirements?
- How do you learn a new technology quickly?
- Describe a time you worked in a team.

## Interview Rules

Applicants must:

- Join on time.
- Use their real name on the meeting platform.
- Keep their camera on unless connectivity prevents it.
- Avoid reading scripted answers.
- Not receive live help from another person.
- Not record the interview without permission.

## Evaluation Criteria

Interviewers score applicants on:

| Area | Weight |
|---|---:|
| Technical fundamentals | 30% |
| Project depth | 25% |
| Problem-solving | 20% |
| Communication | 15% |
| Internship readiness | 10% |

## Common Interview Mistakes

Common mistakes include:

- Not knowing the projects listed on the resume
- Overclaiming skills
- Giving memorized answers without understanding
- Not asking clarifying questions
- Ignoring edge cases
- Being unable to explain tradeoffs
- Not preparing questions for the interviewer
