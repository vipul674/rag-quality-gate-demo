# AI Tools Usage Policy

**Company:** Apex Nimbus Labs  
**Effective Date:** January 1, 2026  
**Policy Owner:** Security and Product Governance Team

## 1. Purpose

This policy defines responsible use of AI tools at Apex Nimbus Labs.

## 2. Approved Use Cases

Employees may use approved AI tools for:

- Drafting internal documents.
- Summarizing non-sensitive information.
- Brainstorming ideas.
- Writing sample code.
- Creating test data that does not contain real personal data.
- Improving productivity workflows.

## 3. Prohibited Use Cases

Employees must not use AI tools to:

- Upload confidential customer data unless the tool is approved for that data class.
- Generate misleading, fraudulent, discriminatory, or harmful content.
- Make final employment, financial, legal, medical, or compliance decisions without human review.
- Bypass security controls.
- Create impersonations without authorization.
- Submit AI-generated work as final without review.

## 4. Human Review

AI outputs must be reviewed by a human before being used in customer-facing, legal, financial, technical, or operational materials.

## 5. Data Classification

Employees must follow the company data classification policy before entering information into AI tools.

| Data Type | AI Tool Use |
|---|---|
| Public data | Allowed in approved tools |
| Internal data | Allowed in approved enterprise tools |
| Confidential data | Requires explicit approval |
| Regulated personal data | Restricted unless approved by Legal and Security |

## 6. Accountability

Employees remain responsible for work produced with AI assistance.

## 7. Reporting Issues

Report hallucinations, security concerns, biased outputs, or accidental data exposure to the Security team.
